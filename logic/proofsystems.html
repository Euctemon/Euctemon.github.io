Writing proofs

Axiomatic systems

In the previous chapter, we discussed a way how to check satisfiability of a formula in a given language. We noted that for an infinite model, the truth tables come short. So what do mathematicians do instead? They write proofs. A proof is essentially a sequence of formulas. We start with the so called axioms and then use rules for constructing new formulas out of old ones, hoping that at the end we derive the formula we want to prove. The rules are commonly know as inference rules and together with set of axioms constitute an axiomatic system.

Axiomatic systems have a long history. Using set of axioms and inference rules dates back to Euclid and his elements. Euclid called the axioms of geometry as postulates. But the proper study of axiomatic systems can be said to begin with Frege and Hilbert. They were not only using logic for mathematics, but they were interested in the axiomatic systems itself. What follows is a version of an axiomatic system for propositional logic Hilbert was using.

Axioms
  1. \(\mathfrak{a} \to (\mathfrak{b} \to \mathfrak{a})\)
  2. \((\mathfrak{a} \to (\mathfrak{b} \to \mathfrak{c})) \to ((\mathfrak{a} \to \mathfrak{b}) \to (\mathfrak{a} \to \mathfrak{c}))\)
  3. \(\mathfrak{a} \to (\mathfrak{b} \to \mathfrak{a} \land \mathfrak{b})\)
  4. \(\mathfrak{a} \land \mathfrak{b} \to \mathfrak{a}\)
  5. \(\mathfrak{a} \land \mathfrak{b} \to \mathfrak{b}\)
  6. \(\mathfrak{a} \to (\mathfrak{b} \lor \mathfrak{a})\)
  7. \(\mathfrak{b} \to (\mathfrak{b} \lor \mathfrak{a})\)
  8. \((\mathfrak{a} \to \mathfrak{c}) \to (\mathfrak{b} \to \mathfrak{c}) \to (\mathfrak{a} \lor \mathfrak{b} \to \mathfrak{c})\)
  9. \(\lnot \lnot \mathfrak{a} \to \mathfrak{a}\)
Rules
  • If we can prove \(\mathfrak{a}\) and \(\mathfrak{a} \to \mathfrak{b}\), we can prove \(\mathfrak{b}\).

We see that Hilbert's axiomatic system has only one rule called modus ponens. Philosophers use this implication rule for many types of logic and they sometimes call it modul ponendo ponens. This can be translated as method that by affirming, affirms. The rules we use for producing new formulas out of the proven ones are called inference rules, because we infer new things from old ones and the axioms are often called axiom schemas. This is because \(\mathfrak{a}, \mathfrak{b}\) and \(\mathfrak{c}\) are not concrete formulas, but rather symbols which can be substitued for any formula we want. We can for example instatiate the first axiom schema as \(p \to (q \to p)\) or \((p \land q) \to ((r \lor s) \to (p \land q))\) where \(p, q, r\) and \(s\) are variables for the propositional logic.

The interesting thing is that these axiom schemata were chosen so that every one of them produce tautology. You can check that with truth tables. Before we look at how we use such axiomatic system, we write down the definition of a proof. This definition is used for any language and not just for the language of propositional logic.

Proof is a sequence \(\Psi_0,\ldots,\Psi_n\) of formulas such that for every \(k < n + 1\), either \(\Psi_k\) is a formula in context \(\Gamma\), an axiom schema or there are indices \(i_0,\ldots,i_l< k\) such that \(\Psi_k\) follow from \(\Psi_{i_0}, \ldots, \Psi_{i_l}\) by some inference rule. We say that \(\Psi_n\) is the conclusion and the sequence \(\Psi_0,\ldots,\Psi_n\) with hypotheses in \(\Gamma\) is a proof of \(\Psi_n\). We write \(\Gamma \vdash \Psi\) if there exists a proof of \(\Psi\) given the context \(\Gamma\).

We notice that the definition features something called context. This is a set of formulas which we suppose are true in the context of a given proof. It is not always the case that \(p \lor q\) is true, but we can add it to the context and then build proofs upon this assumption. Suppose we choose a context made of \(c \lor b\), \(c \to d\) and \(b \to d\). The variable \(c\) stands for car, \(b\) for bus and \(d\) for staying dry.

\(\{c \lor b, c \to d, b \to d\} \vdash d\).
  1. Obtain \((b \to d) \to (c \to d) \to ((b \lor c) \to d)\) by instantiating the axiom schema 8.
  2. Obtain \((c \to d) \to ((b \lor c) \to d)\) by modus ponens using the context formula \(b \to d\) and the formula obtained in step 1.
  3. Obtain \((b \lor c) \to d\) by modus ponens using the context formula \(c \to d\) and the formula obtained in step 2.
  4. Finally obtain \(d\) by modus ponens using the context formula \(b \lor c\) and the formula obtained in step 3.

Supposing we have either a car or a bus at our disposal and that both car and bus will keep us dry, we concluded that we would stay dry. The context is important. We can't prove that we stay dry just because we wish so.

So far, we used concrete formulas in our proofs. Now let's take a formula \((p \lor q) \land r\). We would like to prove that it is equivalent to \(r \land (p \lor q))\). What we can do is that we can check whether \(s \land r\) is equivalent to \(r \land s\) and treat the variable \(s\) simply as a resulting truth value of the subformula \(p \lor q\). If someone gave is the formula \(s \land r\), we wouldn't know if the rewrite was made. This way, we blur the line between concrete formulas and schemata. Let's see an example.

\(\{\mathfrak{a} \land \mathfrak{b}\} \vdash \mathfrak{b} \land \mathfrak{a}\).
  1. Obtain \(\mathfrak{a}\) by modus ponens using the context and axiom schema 4.
  2. Obtain \(\mathfrak{b}\) by modus ponens using context and axiom schema 5.
  3. Obtain \(\mathfrak{a} \to (\mathfrak{b} \land \mathfrak{a})\) by modus ponens using \(\mathfrak{b}\) and axiom schema 3.
  4. Obtain \(\mathfrak{b} \land \mathfrak{a}\) by modus ponens using \(\mathfrak{a} \to (\mathfrak{b} \land \mathfrak{a})\) and \(\mathfrak{a}\).

We learned something about the conjugation symbol. It does not matter if we swap the position of formulas \(\mathfrak{a}\) and \(\mathfrak{b}\). We say that the conjunction is commutative. Let's prove something seemingly trivial.

\(\varnothing \vdash \mathfrak{f} \to \mathfrak{f}\).
  1. Obtain \(\mathfrak{f} \to (\mathfrak{f} \to \mathfrak{f})\) from the the axiom schema 1.
  2. Obtain \(\mathfrak{f} \to ((\mathfrak{f} \to \mathfrak{f}) \to \mathfrak{f})\) from the axiom schema 1.
  3. Obtain \((\mathfrak{f} \to ((\mathfrak{f} \to \mathfrak{f}) \to \mathfrak{f})) \to ((\mathfrak{f} \to (\mathfrak{f} \to \mathfrak{f})) \to (\mathfrak{f} \to \mathfrak{f}))\) from the axiom schema 2.
  4. Obtain \(\mathfrak{f} \to (\mathfrak{f} \to \mathfrak{f})) \to \mathfrak{f} \to \mathfrak{f}) \) by modus ponens using formulas from steps 2 and 3.
  5. Obtain \(\mathfrak{f} \to \mathfrak{f} \) by modus ponens using formulas from steps 4 and 1.

First thing you should note is that the context is empty. This means that the formula is true without any supplemental assumtions, which makes such a formula a tautology. You might ask why isn't this formula an axiom. It seems useful to know it from the start. We could add \(\mathfrak{f} \to \mathfrak{f}\) to the set of axiom schemata and nothing would break. But we strive to have a set of axioms which is in some sense minimal. This means that no axiom in the set is provable using the other axioms.

The following theorem summarizes some important properties of the proof system. Proofs of these statements follow easily from the definition of a proof and the assumption that the rules of the axiomatic system contain modus ponens.

The set of consequences

Suppose \(\Gamma\) is a set of formulas. The set of consequences is the set of formulas provable from \(\Gamma\) and it is inductively generated by these rules

  • If \(A \in \Gamma\), then \(\Gamma \vdash A\)
  • If \(A\) is an axiom schema, then \(\Gamma \vdash A\)
  • If \(\Gamma \vdash A\) and \(\Gamma \vdash A \to B\), then \(\Gamma \vdash B\)

One of the important property of this proof system is the so-called deduction theorem. It enables us to move formulas from the context into the conclusion and shows a connection between context and implications.

Fix a context \(\Gamma\) and a formula \(\mathfrak{f}\). If \(\Gamma \cup\{\mathfrak{f}\} \vdash \mathfrak{g}\) then \(\Gamma \vdash \mathfrak{f} \to \mathfrak{g}\).

We fix \(\Gamma\) and \(\mathfrak{f}\), and use the induction on the set of consequences of \(\Gamma \cup\{\mathfrak{f}\}\). This means that if \(\mathfrak{g}\) is in the set of consequences, it has the form of one of the three cases mentioned earlier:

  1. \(\mathfrak{g} \in \Gamma \cup \{\mathfrak{f}\}\)
  2. \(\mathfrak{g}\) is an axiom schema
  3. There is proposition \(\mathfrak{h}\) such that \(\Gamma \cup\{\mathfrak{f}\} \vdash \mathfrak{h}\) and \(\Gamma \cup\{\mathfrak{f}\} \vdash \mathfrak{h} \to \mathfrak{g}\)

Case 1
If \(\mathfrak{g} \in \Gamma\), we know that \(\Gamma \vdash \mathfrak{g}\) from a definition of a proof. Using the first axiom schema we also know that \(\Gamma \vdash \mathfrak{g} \to (\mathfrak{f} \to \mathfrak{g})\). The conclusion \(\Gamma \vdash \mathfrak{f} \to \mathfrak{g}\) is then obtained by modus ponens. If \(\mathfrak{g}\) is equivalent to \(\mathfrak{f}\), we need to show \(\Gamma \vdash \mathfrak{g} \to \mathfrak{g}\). But we already proved this.

Case 2
If \(\mathfrak{g}\) is an axiom schema, we know that \(\Gamma \vdash \mathfrak{g}\) from definition of proof. We use the first axiom schema to conclude \(\Gamma \vdash \mathfrak{g} \to (\mathfrak{f} \to \mathfrak{g})\) and then use the modus ponens to obtain the conclusion \(\Gamma \vdash \mathfrak{f} \to \mathfrak{g}\).

Case 3
Using the inductive hypothesis we to obtain \(\Gamma \vdash \mathfrak{f} \to \mathfrak{h}\) and \(\Gamma \vdash \mathfrak{f} \to (\mathfrak{h} \to \mathfrak{g})\) from \(\Gamma \cup\{\mathfrak{f}\} \vdash \mathfrak{h}\) and \(\Gamma \cup\{\mathfrak{f}\} \vdash \mathfrak{h} \to \mathfrak{g}\). We use the second axiom schema to obtain \(\Gamma \vdash (\mathfrak{f} \to (\mathfrak{h} \to \mathfrak{g})) \to ((\mathfrak{f} \to \mathfrak{h}) \to (\mathfrak{f} \to \mathfrak{g}))\). Using the modus ponens twice, we conclude \(\Gamma \vdash \mathfrak{f} \to \mathfrak{g}\).

Theories

When someone speaks about a theory of groups, or theory of lattices, they automatically assume one works with first order logic. Apart from using logical connectives, this logic has two quantifiers, the so-called existential \((\, \exists\,\)\) and universal \((\,\forall\,\)\) quantifier. We already know that including logical symbols in language is probably a good idea. Their universal interpretation enables us to write tautologies even though there might not be any tautologies in the set of atomic formulas. Why do we need quantifiers? A formula can be satisfiable, true or tautology. We know that tautology does not depend on the given model. But what happens when we fix a model? Now the formula might be satisfiable or true.

For a single variable in a formula the quantifiers are in correspondence with satisfiability and truth. Fix a language with no functions and only one relation \(r\) and interpret this language in the model with a universe of integers and less than relation. The formula \(\exists x,\: x < 5\) is satisfiable because of number 3, but not true because of number 6. The numbers are the assignments themselves. Explicitly we have assignment functions \(v_1(x) = 3\) and \(v_2(x) = 6\). The situation becomes complicated for more than one variable. What about the atomic formula \(x < y\)? It is not true, but it is satisfiable. This is fine, but rather coarse distinction. For integers we know that for a given number we can always pick a smaller number. Let's look at the formula \(\forall y,\exists x,\: x < y\) and think about the assignment function \(v(x,y)\) for a bit. The function \(v\) assigns integers to the variables \(x\) and \(y\). Let's pretend we can split the assignment function \(v\) into two functions, \(v_x\) and \(v_y\). The function \(v_2\) can assign any number to \(y\), which we model by the universal assignment. The function \(v_1\) then must assign number smaller than \(v_2(y)\). By defining such assignments, we make the formula true.

The use of quantitiers is essential in mathematics. For example if we switch the quantifiers in the last formula as \(\exists y, \forall x,\: x < y\), the formula is no longer true. Without quantifiers, we could only discern whether the formula is true or satisfiable. With quantifiers, we can see how much it is satisfiable. The formula \(\exists y, \exists x,\: x < y\) being true means that we can find two numbers satisfying the inequality. On the other hand \(\forall y, \exists x,\: x < y\) means we can allways find a smaller number \(x\) for any given number \(y\).
